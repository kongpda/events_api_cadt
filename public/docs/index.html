Introduction

Base URL: http://events_api.test

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Request      

POST api/events

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

title   string   

Must not be greater than 255 characters. Example: f

slug   string   

Must not be greater than 255 characters. Example: bmtuq

description   string   

Must not be greater than 65535 characters. Example: Tempore aperiam ratione vel sit aliquid.

address   string   

Must not be greater than 255 characters. Example: vylqfudwbrtgwkt

feature_image   string  optional  

Must not be greater than 255 characters. Example: iochyepqbvsxwfhknrzlporfv

start_date   string   

Must be a valid date. Example: 2025-02-09T15:17:09

end_date   string   

Must be a valid date. Must be a date after start_date. Example: 2123-08-24

category_id   string   

The id of an existing record in the categories table. Example: beatae

user_id   string   

The id of an existing record in the users table. Example: sint

organizer_id   string   

The id of an existing record in the organizers table. Example: iste

participation_type   string   

Example: paid

Must be one of:
  • paid
  • free
capacity   integer   

Must be at least 1. Example: 51

registration_deadline   string   

Must be a valid date. Must be a date before start_date. Example: 2003-09-06

registration_status   string   

Example: open

Must be one of:
  • open
  • closed
  • full
event_type   string   

Example: online

Must be one of:
  • in_person
  • online
  • hybrid
online_url   string  optional  

This field is required when event_type is online or hybrid. Must be a valid URL. Example: http://www.howell.info/eaque-nulla-minus-laboriosam-praesentium

tags   string[]  optional  

The id of an existing record in the tags table.
